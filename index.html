📘 Microsoft Forms setup guide

Let staff self-register via a Microsoft Form. Here's exactly how to set it up for Bank View:

1
Create a new form

Go to forms.microsoft.com → New Form. Title it: "Staff World Cup 2026 – Entry Form". Add a subtitle with the deadline: Thursday 11th June 2026.

2
Add the registration questions

Add a Text question: "Your full name" (Required). Add a second Text question: "Your team name" (Required). Add a Choice question: "Have you paid your £3 entry fee?" – Options: Yes / I'll pay soon.

3
Add score prediction questions (optional)

For each fixture add a Text question like: "Mexico vs South Africa (11 Jun 8pm) – Predicted score e.g. 2-1". Mark as optional so players can submit partial entries. Group by date using Section headers.

4
Share within Bank View

Click Share → under "Send and collect responses" choose "People in my organisation" so only bankview.liverpool.sch.uk accounts can respond. Copy the link and share via email or Teams.

5
Export and import responses

Go to the Responses tab → Open in Excel. Copy the Name and Team Name columns. Paste into the "Import from CSV" box above in format: Name, Team Name (one per line). Click Import.

6
Enter predictions

Switch to the Predictions tab. Select a player, then enter their predicted scores fixture by fixture from the Excel export. Points calculate automatically as results are entered.

💡 Set a reminder to close the form before the first game kicks off: 11th June 2026, 8pm.
☁️ Cloudflare Worker setup guide

One-time setup to enable cloud sync so data persists between sessions and devices.

1
Create a KV namespace

Cloudflare dashboard → Workers & Pages → KV → Create namespace. Name it WC2026_KV. Copy the Namespace ID.

2
Create a new Worker

Workers & Pages → Create application → Worker. Name it wc2026-api. Paste the worker.js code provided alongside this file. Deploy it.

3
Bind the KV namespace

In your Worker → Settings → Bindings → Add binding → KV namespace. Variable name: WC2026_KV. Select your namespace. Save & deploy.

4
Set the API token secret

Worker → Settings → Variables → Add secret. Name: API_TOKEN. Value: a password you choose (e.g. BankView2026!). This protects write access.

5
Configure this app

Click Configure in the Cloud sync card above. Enter your Worker URL (e.g. https://wc2026-api.yourname.workers.dev) and the API token. Click Test & Save.

6
Host on GitHub Pages

Push index.html and admin.html to a GitHub repo. Enable Pages under Settings → Pages → Branch: main. Your public URL is https://yourusername.github.io/reponame/. Share admin.html link with yourself only.
